# Carb Balancer

Carburetor synchronization tool for multi-cylinder engines (up to 4 cylinders).
Reads intake vacuum via pressure sensors, detects the deepest vacuum pull per cylinder per engine cycle,
and displays live balance data on a web UI served directly from the device.

---

## Hardware

| Component | Part | Notes |
|-----------|------|-------|
| Microcontroller | Arduino Nano ESP32 | ESP32-S3, Xtensa LX7 dual-core |
| ADC | ADS1115 (×1) | 16-bit, I2C address `0x48`, 860 SPS |
| Pressure sensor | MPX4250AP (×1–4) | One per cylinder, 5 V supply |
| Signal conditioning | 10 kΩ / 20 kΩ resistor divider | Scales MPX output to ≤ 3.3 V |
| Status LED | Any | Connected to D2 (GPIO5) |

### Wiring

```
MPX4250AP Vout ───── 10 kΩ ──┬── ADS1115 AINx
                             │
                           20 kΩ
                             │
                            GND
```

**ADS1115 → Arduino Nano ESP32**

| ADS1115 | Arduino Nano ESP32 |
|---------|--------------------|
| VDD | 3.3 V |
| GND | GND |
| SCL | A5 (GPIO12) |
| SDA | A4 (GPIO11) |
| ALRT/RDY | A2 (GPIO3) |
| ADDR | GND → I2C address `0x48` |

---

## How it works

1. The ADS1115 samples each active channel in sequence at 860 SPS (≈215 samples/channel/second at 4 cylinders).
2. Each sample is calibrated (offset + gain) then smoothed with an EMA filter.
3. Smoothed values are stored in a per-channel ring buffer with timestamps.
4. RPM is estimated automatically from the interval between successive vacuum minima on the reference channel.
5. The **displayed value** per cylinder is the **minimum EMA reading** within one engine cycle
   (window auto-calculated from RPM + configurable margin). This captures each cylinder's peak vacuum pull.
6. Min readings are broadcast over WebSocket to connected browsers and optionally streamed over USB serial.

## Build & flash

Requires [PlatformIO](https://platformio.org/).

```bash
# Build and flash firmware
pio run --target upload

# Monitor serial output
pio device monitor

# Run unit tests on host (no hardware needed)
pio test --environment native
```

---

## Web UI development

The UI is embedded in firmware as a PROGMEM string — no filesystem (LittleFS/SPIFFS) is used.
During development, edit files in `web/` and test locally without flashing:

```bash
# 1. Start local WebSocket mock (streams simulated vacuum data)
python tools/mock_server.py

# 2. Open web/index.html in a browser — connects to the mock server

# 3. When the UI is ready, build it into the firmware header
python tools/build_web.py

# 4. Recompile and flash once
pio run --target upload
```

## Settings (via web UI)

All settings are saved to NVS and restored on boot.

| Setting | Default | Description |
|---------|---------|-------------|
| Active cylinders | 4 | 1–4 |
| Reference cylinder | 1 | Cylinder used for RPM detection and balance reference |
| Offset (per channel) | 0 | Zero offset in raw ADC counts |
| Gain (per channel) | 1000 | Gain multiplier ×1000 (1000 = 1.000×) |
| EMA α | 100 | Smoothing factor ×1000 (100 = α 0.1) |
| Window margin | 20 % | Extra margin added on top of one engine cycle for min-tracking |
| Update interval | 100 ms | WebSocket broadcast interval |
| WiFi SSID | `CarbBalancer` | AP network name |
| WiFi password | `balance1` | AP password |
